What is a build tool

A build tool basically makes sure that your project ends up as an executable file (.jar / .war).

It helps you automate repetitive tasks such as:

  • ♥ compile source code into binary
  • ♥ package the binary code into executable files
  • ♥ running tests
  • ♥ download and manage dependencies
  • ♥ deploy your project onto production system

Wait. Dependencies?

Dependencies are the libraries, plugins, tools, frameworks that your web app is using (depending on).

These dependencies have dependencies of their own.

Sometimes they will depend on different and incompatible versions of the same library.

Fixing one dependency will then break another.

This is professionally referred to as

Dependency Hell

You do not want to go there

AJAX. A concept

AJAX is not a tool, language or technology, but a concept.

This concept combines technologies to enable asynchronous communication with the server.

Little bits of information are being communicated with the server only if needed.

Your website will be quicker and your users will not have to wait for a full page reload each time.

When to use AJAX

AJAX was made famous by Google, they use it for google Suggest, Gmail and Google Maps.

It is used by populair website such as Facebook and Youtube.

Websites with impatient users, expecting high amounts of user specific data.

Dynamic content

Lot of user action

Investigate SEO implications!

For a static website you better look at other options to enhance performance

Think about caching, local storage, offline storage of data instead
